# Session: Revisao modo compact de relatorio

## Objetivo
Analisar os reports batch gerados nos 3 modos (compact/full/minimal), identificar pontos de melhoria no modo compact para reduzir tokens sem perder informacoes relevantes para analise de performance SQL.

## O que foi feito
- Analise comparativa de 42 reports em 3 modos (126 arquivos) — diagnosticou que 48% dos reports tinham compact = full (0% economia)
- Identificou 6 problemas principais: SQL text sempre incluso, indices nao filtrados, R1/R2 com tabelas vazias em planos estimados, PX infra nao colapsado, views repetidas, Column Projection passando no compact
- Implementou **M1**: flag `--show-sql` (default off no compact) — omite bloco SQL completo, mantendo apenas metadados
- Implementou **M2**: flag `--show-all-indexes` (default off) — filtro inteligente de indices por colunas do SQL (WHERE/JOIN/ORDER/GROUP) + plano
- Implementou **M3**: fix R1/R2 para planos estimados — omite tabela de detalhes com `? | 0 | 0`
- Implementou **M6**: strip Column Projection Information no modo compact
- Atualizou docs: README.md, POWER.md, architecture.md
- Atualizou testes: 526 testes passando

## Decisoes tomadas
- `--show-sql` e `--show-all-indexes` sao flags ortogonais ao verbosity (nao niveis) — controlam conteudo especifico
- Por padrao no compact: sem SQL text, sem indices irrelevantes, sem Column Projection, sem tabela de detalhes em R1/R2 estimados
- `--verbosity full` sempre mostra tudo (ignora os flags)
- Filtro de indices combina dois criterios: colunas do SQL + referencias no plano (R9)
