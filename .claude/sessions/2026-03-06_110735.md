# Sessao: Testes de Integracao com Oracle Docker

## Objetivo

Criar infraestrutura de testes de integracao com Oracle real via Docker, validando o SqlMentor contra Oracle XE 21c com tabelas modelo e cobertura dos principais fluxos (connection, collector, EXPLAIN PLAN, inspect V$SQL, report).

## O que foi feito

- Criada branch `feat/oracle-integration-tests`
- Docker Compose com Oracle XE 21c (`gvenzl/oracle-xe:21-slim`) + healthcheck + tmpfs
- Script de init (`01_setup.sh`) que conecta ao PDB (XEPDB1) como SYSDBA e cria: user read-only, 3 tabelas (DEPARTMENTS 10 rows, EMPLOYEES 1000 rows, ORDERS 5000 rows), 1 VIEW, 1 FUNCTION, seed data, gather stats
- 49 testes de integracao em 5 arquivos cobrindo: conexao/privilegios, coleta de metadata (stats, columns, indexes, constraints, DDL, batch, view expansion), EXPLAIN PLAN (estimado, join, binds, cleanup, runtime), inspect V$SQL (sql_text, runtime_plan, ALLSTATS, wait events), report end-to-end (markdown, JSON, compact vs full)
- Testes unitarios (541) continuam passando sem alteracao (`--ignore=tests/integration` no addopts)
- CI atualizado com job `integration-oracle` (docker run + wait + pytest)
- Documentacao de setup Docker para WSL2 Ubuntu
- **Resultado: 49 passed, 0 failed, 541 unit tests intactos**

## Decisoes tomadas

- **Oracle XE 21c** como versao inicial (thin mode, sem Instant Client, sem diferencas estruturais vs 19c para o que o SqlMentor usa). 11g e 23ai Free para fases futuras
- **Sem APP_USER do gvenzl** — scripts `.sql` com APP_USER rodam no CDB, nao no PDB. Solucao: unico shell script que conecta explicitamente ao PDB via `SYS@XEPDB1 AS SYSDBA`
- **Privilegios minimos validados**: CREATE SESSION, SELECT_CATALOG_ROLE, SELECT ANY TABLE, SQLMENTOR_EXEC_ROLE (mesmo modelo do `scripts/oracle_create_user.sql`). CREATE TABLE/VIEW/PROCEDURE concedidos apenas durante init e revogados apos criar objetos
- **Isolamento de testes**: `addopts = "--ignore=tests/integration"` no pyproject.toml garante que `task test` nao roda integracao. `task test-oracle` para rodar apenas integracao
- **Session-scoped connection**: uma unica conexao Oracle para todos os testes de integracao (evita overhead de reconexao)
- `.gitattributes` para forcar LF em `.sh` (evitar `^M: bad interpreter` no container Linux)
