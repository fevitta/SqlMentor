# Sessão: Claude Code Agent + Skill para SqlMentor

## Objetivo

Criar um agente customizado e slash commands para o Claude Code que permitam usar o CLI `sqlmentor` de forma integrada — coletando contexto Oracle e produzindo análises de tuning diretamente pelo Claude.

## O que foi feito

- Criado agente customizado em `.claude/agents/sqlmentor.md` com persona de DBA Oracle sênior
- Criado `settings.json` com MCP server e permissões auto-approve para comandos sqlmentor
- Criados 5 slash commands (`/analyze`, `/inspect`, `/tuning`, `/doctor`, `/parse`) — depois removidos por serem redundantes com o agente
- Removida duplicação de documentação de flags/parâmetros do agente — agora usa `sqlmentor --help` dinamicamente
- Adicionada instrução "CLI primeiro, MCP nunca" ao agente
- Testado o agente com 5 queries reais do `sql_candidates.csv` — 5 relatórios gerados com sucesso

## Decisões tomadas

1. **Agente > Slash commands**: o agente cobre todos os fluxos (analyze, inspect, tuning, doctor) sem precisar de commands separados
2. **CLI > MCP**: o agente sempre usa o CLI via Bash (gera arquivos em `reports/`, mostra progresso) em vez do MCP server
3. **Sem duplicação de docs**: flags e parâmetros não são hardcoded no agente — ele descobre via `--help` para evitar dessincronização com cli.py/mcp_server.py/POWER.md/README.md
4. **Metodologia de análise embutida**: o agente carrega a metodologia DBA sênior (mesma do steering `analysis.md` do Kiro Power) diretamente no seu prompt
5. **Model sonnet + maxTurns 30**: balanceamento entre velocidade e capacidade para fluxos multi-step
